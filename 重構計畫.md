# 重構優先順序

## Summary
先處理會影響實盤安全與每日流程可用性的問題，再穩定設定與品質基線，最後才拆研究大檔與補實驗可信度。排序依據是：下單風險 > 相容性 > 測試保護 > 研究重構收益。

## Priority Order
1. **P0：修 live/dry-run guard 斷點**
   - 先修 `trade_guard.py` 仍呼叫 `CMoneyRPA.inventory()` 的 runtime 問題，統一改走 `get_account_status()["inventory"]` 或同一個 adapter。
   - 確認 `pending_buys`、sell-only、正式 `--execute` 都共用同一套 pre-trade guard。
   - 產出並驗證 `dry_run_diff`，live 前必須檢查 signal TTL、aid match、macro guard、risk limits。

2. **P1：補測試與品質基線**
   - 固定每階段必跑：`python -m compileall -q -x env .`、`python -m unittest discover -s tests -v`、`ruff check .`。
   - 先加 guard 測試，覆蓋 live flag false、aid mismatch、expired signal、macro WARN/CRITICAL、dry-run diff 缺失、曝險超標。
   - `git` 目前在環境中不可用，短期先不把 git 狀態納入驗收命令。

3. **P2：整理設定層，不大搬家**
   - 保留現有 `settings.py`，補足缺少的 research/evaluation/live/paths/risk_limits 使用點。
   - CLI 參數維持相容；內部改成先載入 `load_settings()`，再由 CLI 覆寫。
   - 不新增 `config/` 目錄，除非後續設定膨脹到單檔難以維護。

4. **P3：拆 `data_loader.py`**
   - 先拆資料管線，因為它同時影響 training、evaluation、walk-forward。
   - 分成 market fetch、single-asset feature engineering、multi-asset enrichment。
   - 保留 `fetch_multi_asset_data(...)` facade，新增 `FeatureSchema` 或等價 schema 檢查，避免 observation shape 漂移。

5. **P4：拆 `walk_forward.py` 與研究編排**
   - 拆 period planning、train/eval orchestration、artifact persistence、metrics aggregation。
   - 共用 `ResearchSettings` 的日期、seed、algo、cash mode、feature path、result path 規則。
   - 保留既有 CLI、檔名與 `results_dir` layout。

6. **P5：整理 `experiment_report.py` 與 promotion gate**
   - 報告只讀 results，不碰 live signal 或下單狀態。
   - 加 baseline、ablation、cost/slippage sensitivity、weak-period、seed dispersion、MDD/turnover gate。
   - 先支援舊結果格式與 `_with_features` 結果並存。

7. **P6：最後才拆 `cmoney_rpa.py` 深層結構**
   - 在 guard 已穩、測試已補後，再拆 signal validation / rebalance planning、CMoney API client、CLI glue。
   - 這一步風險高，避免早期大改 live path。

## Public Interfaces
- 必須保留：`fetch_multi_asset_data(...)`、`train_portfolio.py` CLI、`walk_forward.py` CLI、`evaluate_portfolio.py` CLI、`daily_trade_runner.py`。
- 正式化：`RiskLimits`、`ResearchSettings`、`FeatureSchema`、pre-trade guard、`dry_run_diff` JSON。
- 短期不改模型 zip、`results_dir`、現有 signal/report 檔名格式。

## Test Plan
- P0/P1 後：完整 unittest、compileall、ruff，外加 dry-run command，不送實單。
- P3 後：feature schema、缺欄位 fail-fast、overnight feature 無 lookahead。
- P4 後：period clamp、skip/overwrite decision、result path、metrics aggregation。
- P5 後：baseline 可產生、ablation 可比較、slippage 改變核心指標、promotion gate 會明確阻擋不合格結果。

## Assumptions
- 每一步完成後 `daily_trade_runner.py` 仍可 dry-run。
- 不改 reward、模型超參數、ticker universe、walk-forward periods 預設。
- 優先採現有 `settings.py`，不先引入更複雜的 config 架構。
