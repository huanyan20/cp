# CP 專案總覽

> Strategy snapshot: 2026-06-16
> Current research direction: SL-first, SAC research-only.

## 1. Current Decision

本專案目前不再把 SAC / PPO 強化學習模型作為上線主線。最新 SAC 結果顯示仍有 alpha 雛形，但存在三個不可忽視的問題：seed 間差異大、績效集中於少數期間、cash-enabled 版本容易退化成長期全現金。單一 seed 或單一期間的高報酬不得作為 live promotion 依據。

後續主線改為：

```text
SL signal -> rule allocator -> risk gate -> trade_guard -> CMoney RPA
```

SAC 保留為低成本研究線，只用於 reward/action/allocator ablation，不再預設完整 300K 多 seed promotion。

## 4. Promotion Rule

任何模型要進 live，至少需要同時滿足：

- OOS Sortino 達標且跨 period 穩定。
- Max drawdown 在風險預算內，不只平均值漂亮。
- Turnover 經過 fee/tax/slippage 後仍可接受。
- Cash 行為不能是全現金坍縮，也不能以 0 報酬低回撤偽裝穩健。
- 弱期如 2024H2、2025H1、2025H2 不能靠單一強期掩蓋。

## 6. Live Operational Reliability (Dry-run Phase)

> Dry-run validation is observation only. CMoney RPA remains blocked unless the
> active candidate gate reports `core` or `full` approval.

目前 SL h10 進入「上線前防呆與監控」階段。核心模型（`labels.py`, `signal_generator.py`, `rule_based_allocator.py`）**已凍結**，暫不再為追求更高 return 而調參。任何後續核心更動皆須視為新候選並重跑 3-seed gate。

### 觀察重點 (Watch Items)
- **Daily Dry-run**：每天只產生 signal + trade_guard diff，不送單。需觀察：
  - target weights 是否劇烈跳動
  - 是否過度集中同一批股票
  - macro guard 是否合理降風險
  - total exposure 是否過高
  - signal 會不會因資料延遲或 TTL 出問題
- **Seed44 MDD**：Seed 44 的 MDD 較貼近閥值，需列為 watch item。若 live dry-run 出現過度曝險，優先調降 live allocator 的 exposure cap，而非重訓模型。
